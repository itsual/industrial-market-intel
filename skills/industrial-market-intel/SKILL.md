# Industrial Market Intelligence Agent

## When to Use This Skill

Load this skill when the user asks for any of the following:

- "Give me today's market news on water treatment / ZLD / OARO / membranes"
- "What are competitors doing in the ion exchange / specialty chemicals space?"
- "Send me a weekly market intelligence briefing"
- "Any new regulations on industrial wastewater in India / EU / US?"
- "Monitor [company name] for new announcements"
- "Summarize recent M&A or funding activity in water tech"
- Scheduled daily/weekly briefings on industrial market sectors

## Configuration

Before first use, the user should set up a `config.json` in the skill directory (or workspace root). A template is provided below.

```json
{

  "regions": ["India", "Southeast Asia", "Middle East", "Europe"],
```

## Instructions

### Step 1 — Determine Request Type

Identify what the user wants:
- **On-demand briefing**: Run immediately and return/send the report.
- **Scheduled briefing**: Set up a cron job using OpenClaw's scheduler.
- **Competitor watch**: Focus the search on a named company.
- **Regulatory scan**: Focus on policy, regulation, and standards news.

### Step 2 — Run News Fetch

Execute `scripts/fetch_news.py` with the appropriate topic and region filters.

```bash
python scripts/fetch_news.py --topics "water treatment,ZLD,OARO,ion exchange" --regions "India,Global" --days 1
```

Parameters:
- `--topics`: Comma-separated list of search topics (from config or user input)
- `--regions`: Geographic filter
- `--days`: Lookback window in days (1 for daily, 7 for weekly)
- `--competitors`: Optional comma-separated competitor names for focused watch
- `--output`: Output file path (default: `/tmp/raw_news.json`)

The script returns a JSON array of news items with fields: `title`, `source`, `url`, `date`, `summary`, `relevance_score`.

### Step 3 — Analyze and Score

Review the fetched news items and:
1. Filter out items with `relevance_score < 0.4`
2. Categorize each item into one of: `Market Trends`, `Competitor Activity`, `Regulatory Updates`, `Technology Developments`, `M&A / Funding`, `Contracts & Projects`
3. Flag high-priority items (competitor wins, major regulatory changes, funding > $10M)
4. Add a 1–2 sentence AI commentary for each high-priority item explaining the strategic implication

### Step 4 — Format Report

Execute `scripts/format_report.py` to produce the final markdown report.

```bash
python scripts/format_report.py --input /tmp/raw_news.json --output /tmp/market_briefing.md --period "Daily" --date "$(date +%Y-%m-%d)"
```

The report structure:
```
# Industrial Market Intelligence Briefing
## [Daily/Weekly] — [Date]

### Executive Summary
2–3 sentences on the most important developments.

### Market Trends
...

### Competitor Activity
...

### Regulatory Updates
...

### Technology Developments
...

### M&A / Funding
...

### Contracts & Projects
...

---
*Generated by industrial-market-intel OpenClaw skill*
```

### Step 5 — Deliver or Save

**If delivering via chat channel:**
```bash
python scripts/send_briefing.py --input /tmp/market_briefing.md --channel telegram
```

**If saving to workspace:**
Save the report to `~/openclaw/workspace/briefings/market_intel_YYYY-MM-DD.md`

**If returning inline:**
Read `/tmp/market_briefing.md` and return the content directly in the chat response.

### Step 6 — Set Up Schedule (if requested)

If the user wants recurring briefings, register a cron using OpenClaw's scheduler:

```
Every day at 07:00 IST: run industrial-market-intel skill with default config
Every Monday at 08:00 IST: run industrial-market-intel skill with --days 7 for weekly summary
```

Confirm the schedule with the user before activating.

## Error Handling

- If no news is found for a topic, return a "No significant updates" entry for that category — do not skip the category entirely.
- If a web search fails, log the error and retry once before reporting the failure to the user.
- If the delivery channel is unavailable, save the report to workspace and notify the user via available channel.

## Privacy & Data Notes

- All data is fetched from public sources (web search, RSS feeds, news APIs).
- No proprietary data is transmitted externally.
- Reports are stored locally in the OpenClaw workspace.
